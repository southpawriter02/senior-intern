# v0.5.3e – Working Directory Synchronization

> Bi-directional directory sync, OSC 7 parsing, terminal/explorer integration.

## Overview

This version implements working directory synchronization between terminal and file explorer.

## Changes

### Added

- **Event Models:**
  - `DirectoryChangeSource` enum (Shell, Osc7, Api, ExplorerSync, WorkspaceSync)
  - `DirectoryChangedEventArgs` class
  - `Osc7EventArgs` class

- **`IWorkingDirectorySyncService`** interface with:
  - Events: TerminalDirectoryChanged, ExplorerDirectoryChanged
  - Query: GetTerminalDirectoryAsync, IsAutoSyncEnabled
  - Commands: ChangeTerminalDirectoryAsync, SyncTerminalToExplorerAsync, SyncExplorerToTerminalAsync
  - Configuration: SetAutoSync
  - Workspace Linking: LinkToWorkspace, UnlinkFromWorkspace
  - OSC Integration: ProcessOsc7

- **`WorkingDirectorySyncService`** implementation with:
  - OSC 7 escape sequence parsing (file:// URI)
  - WSL path translation (/mnt/c → C:\)
  - Auto-sync based on DirectorySyncMode settings
  - Workspace linking for grouped terminals
  - ConcurrentDictionary for thread-safe session tracking
