# v0.3.2f: Keyboard & Interactions - Detailed Changelog

## Overview

Documentation of keyboard shortcuts, interaction patterns, and user flows for the file explorer.

> [!NOTE]
> No new files are created in v0.3.2f. All implementation was done in v0.3.2d.

## Keyboard Shortcuts

| Shortcut | Action | Context |
|----------|--------|---------|
| `↑` / `↓` | Select prev/next | TreeView |
| `→` | Expand folder | Folder selected |
| `←` | Collapse folder | Folder selected |
| `Enter` | Open / Toggle | Item selected |
| `F2` | Start rename | Item selected |
| `Delete` | Delete item | Item selected |
| `Escape` | Cancel rename | Rename mode |
| `Ctrl+C` | Copy path | Item selected |
| `Ctrl+Shift+C` | Copy relative | Item selected |
| `Ctrl+N` | New file | Any |
| `Ctrl+Shift+N` | New folder | Any |
| `Ctrl+R` | Refresh | Any |

## Inline Rename Flow

1. **Trigger**: F2 or context menu → Rename
2. **Edit mode**: TextBox replaces TextBlock
3. **Commit**: Enter or focus loss
4. **Cancel**: Escape key

### Validation Rules
- ✅ Valid: alphanumeric, dashes, underscores, dots, hidden files
- ❌ Invalid: empty, `/`, `\`, `:`, `*`, `?`, `<`, `>`, `|`, `"`

## Delete Confirmation

- Files: "Delete 'filename.ext'?"
- Folders: "Delete 'foldername' and all its contents?"
- Default button: Cancel (safety)

## Focus Management

1. TreeView (primary)
2. Filter TextBox (Tab navigation)
3. Rename TextBox (inline, auto-focus)
