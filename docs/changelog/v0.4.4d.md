# v0.4.4d Changelog - Tree ViewModels

**Released**: 2026-01-16

## Summary

Added presentation layer ViewModels for multi-file proposals with hierarchical tree building, selection management, and validation display.

## Features

### Enums
- **`SelectionState`** - None/Some/All states for directory tri-state checkboxes
- **`TreeNodeSelectionBehavior`** - Selection propagation options (Independent/CascadeToChildren/CascadeToParent/CascadeBoth)
- **`TreeBuildingOptions`** - Configuration record for tree building

### Services
- **`ITreeBuildingService`** - Interface for building hierarchical trees from flat operations
- **`TreeBuildingService`** - Implementation that groups files by directory, creates intermediate nodes, and sorts alphabetically
- **`TreeNode`** - Data model for tree nodes

### ViewModels  
- **`FileOperationItemViewModel`** - Tree item ViewModel for proposals with icon resolution, validation display, and status tracking
- **`FileTreeProposalViewModel`** - Main proposal ViewModel with selection management, validation integration, and apply operations

## Tests

- `TreeBuildingServiceTests` - 15 tests covering tree building, flattening, and searching
- `FileOperationItemViewModelTests` - 33 tests covering computed properties, factory methods, and state management

## Dependencies

- v0.4.4a: Core models (FileTreeProposal, FileOperation)
- v0.4.4b: FileTreeParser
- v0.4.4c: FileTreeProposalService
